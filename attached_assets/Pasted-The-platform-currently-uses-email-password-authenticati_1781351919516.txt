The platform currently uses email/password authentication. Add "Sign in with
Google" and other social login options to both the signup and login pages,
on top of the existing auth. Typebot CE is built on NextAuth.js (Auth.js) —
use its built-in OAuth provider support rather than building this from
scratch.

1. **Providers to add:**
   - Google (priority #1)
   - Facebook
   - GitHub
   - Microsoft / Azure AD (useful for B2B clients using Office 365)
   - Apple (Sign in with Apple — note this requires an Apple Developer
     account and has stricter requirements; build the provider config slot
     for it, but treat it as optional if time-constrained)

   For each provider, configure the NextAuth provider with clientId/
   clientSecret read from the database (PlatformSettings/OAuth config table),
   with env vars only as a fallback for local dev.

2. **Super Admin Panel integration:**
   - Add a new "Authentication / Social Login" section to the existing Super
     Admin Panel where I can, per provider:
     - Toggle it on/off (only show enabled providers as buttons on the
       login/signup pages)
     - Enter Client ID and Client Secret (encrypted at rest, using the same
       approach as the existing Stripe/PayPal credential storage)
     - See the exact redirect/callback URL to paste into each provider's
       developer console (e.g. https://yourapp.com/api/auth/callback/google)
   - Include a short on-screen guide/links for where to create OAuth
     credentials in Google Cloud Console, Meta for Developers, GitHub
     Developer Settings, Azure Portal, and Apple Developer — so I know
     exactly where to go to get each Client ID/Secret

3. **Account creation & linking flow:**
   - First-time social login automatically creates a User + a new
     Organization for them, using the same trial-plan/onboarding logic as
     the existing email signup, pulling name, email, and profile picture
     from the provider
   - If a user signs up with email/password first and later uses "Sign in
     with Google" using the SAME email address, link the accounts (don't
     create a duplicate Organization) — only auto-link if the provider
     confirms the email is verified; otherwise show an "account already
     exists, please log in with your password and link this provider from
     Settings" message
   - Add a "Connected Accounts" section in user Settings where users can see
     which social providers are linked and link/unlink additional ones
     (always require at least one login method to remain — i.e. don't allow
     unlinking the only way to sign in unless a password is set)

4. **UI:**
   - On the existing login and signup pages, render "Continue with
     Google / Facebook / GitHub / Microsoft / Apple" buttons ABOVE the
     email/password form, only showing buttons for providers currently
     enabled in Super Admin settings, with an "or" divider before the email
     form
   - Use each provider's official branding guidelines for button styling
     (correct logos/colors), since most providers require this for approval

5. **Security considerations:**
   - Verify email_verified=true from the OAuth provider before treating the
     email as trusted for account linking
   - Confirm the existing multi-tenant isolation applies identically to
     socially-created accounts (new Organization, scoped data, correct
     default plan assignment)
   - Rate-limit and log failed OAuth callback attempts
